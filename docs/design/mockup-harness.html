Skill: commit

建立標準化的 Git Commit，遵循 Conventional Commits 規範，並自動加入協作者署名。

使用方式

/commit -m "feat(auth): add OAuth2 login flow"

規則

  • 遵循 Conventional Commits 格式（feat / fix / chore / docs / refactor）
  • 自動加入 Co-Authored-By: Claude 署名
  • 禁止使用 --no-verify 旗標
  • Commit 前必須確認 git diff --staged 不為空
  • Message 長度不超過 72 字元

範例輸出

feat(harness): add skill toggle endpoint Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>

禁止行為

  • 不得 amend 既有 commit（除非使用者明確要求）
  • 不得對 main/master 分支直接 push
  • 不得略過 pre-commit hook

kill-port-blocker PreToolUse
Matcher Bash(kill|taskkill)
● 已啟用

Hook: kill-port-blocker

在 Bash 工具執行 kill 或 taskkill 指令前攔截，防止 Agent 意外終止關鍵系統程序。

類型

PreToolUse — 在工具執行前觸發，可阻止執行

Matcher 規則

{"matcher": "Bash(kill|taskkill)"}

處理邏輯

  • 比對指令是否包含 kill 或 taskkill
  • 若比對成功則回傳 block: true 阻止執行
  • 同時記錄觸發事件至 trigger log

Hook 腳本

#!/bin/bash # kill-port-blocker PreToolUse hook echo '{"decision":"block","reason":"kill/taskkill commands are forbidden"}'

# Code Review SOP
v2.1 更新日期: 2026-03-01
## 目的

確保每次程式碼合併前經過系統性的品質審查，防止技術債累積，維護代碼庫的長期可維護性。

## 適用範圍

所有 AgentHub 子專案的 Pull Request，包括功能開發、Bug 修復與重構。Agent 自動提交的程式碼同樣適用。

## 審查流程
### 1. 自動檢查（G3 Gate）
  • TypeScript 型別檢查通過
  • ESLint 無 error（warning 可接受但需記錄）
  • 單元測試覆蓋率 > 70%
  • Build 成功
### 2. 人工審查清單

審查者需確認以下項目，使用 LGTM 標記通過或 NEEDS_REVISION 要求修改：

[ ] 邏輯正確性：核心邏輯是否符合 spec
[ ] 邊界案例：空值、超長輸入、並發情境
[ ] 命名規範：遵循 .knowledge/coding-standards.md
[ ] 錯誤處理：所有 async 操作有 try/catch
[ ] 安全性：無 SQL injection、XSS 風險
[ ] 效能：無明顯 N+1 查詢或記憶體洩漏
## 常見踩坑

警告：不要在 Agent Session 中使用 git push --force，已有 PreToolUse Hook 攔截，但仍需注意避免。
